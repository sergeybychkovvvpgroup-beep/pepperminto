# Getting Started

This guide gets Pepperminto running locally with the default developer setup.

## 1) Install dependencies

```bash
pnpm install
```

## 2) Configure the API

Create or update `apps/api/.env` (or root `.env`) with:

```bash
DATABASE_URL=postgresql://user:pass@localhost:5432/pepperminto
JWT_SECRET=replace-me
```

## 3) Run migrations

```bash
pnpm --filter api prisma migrate deploy
```

## 4) Start the stack

```bash
pnpm dev
```

## Default ports

- `knowledge-base` → `http://localhost:3000`
- `api` → `http://localhost:3001`
- `client` → `http://localhost:3002`
- `docs` → `http://localhost:3003`
- `landing` → `http://localhost:3004`

## First login

If your database has seed data, use the admin credentials documented in your setup or seed script. Otherwise, create your first user through the API or the admin UI.
