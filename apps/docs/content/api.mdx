# API

Pepperminto exposes a REST API under `/api/v1`.

## Base URL

```
http://localhost:3001/api/v1
```

## Authentication

Most routes require a Bearer token:

```bash
curl --request GET \
  --url http://localhost:3001/api/v1/tickets/open \
  --header 'authorization: Bearer <token>'
```

## Swagger

Swagger UI is available at:

- `http://localhost:3001/docs`
- JSON spec: `http://localhost:3001/docs/json`

## Common endpoints

- `POST /auth/login` – login and get a JWT
- `GET /tickets/open` – list open tickets
- `POST /ticket/create` – create a ticket
- `GET /knowledge-base/public` – list published KB entries
