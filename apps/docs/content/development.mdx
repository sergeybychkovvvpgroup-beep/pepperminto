# Development

This guide walks through local development for the Pepperminto monorepo.

## Prerequisites

- Node.js 20+ (22 recommended)
- pnpm 10+
- PostgreSQL 14+

## Install dependencies

```bash
pnpm install
```

## Environment variables

Pepperminto loads environment variables from:

- repo root: `.env`
- per-app: `apps/api/.env`, `apps/client/.env`, etc.

Minimal API values (example):

```bash
DATABASE_URL=postgresql://user:pass@localhost:5432/pepperminto
JWT_SECRET=replace-me
```

## Database

Apply migrations for the API schema:

```bash
pnpm --filter api prisma migrate deploy
```

## Run the full stack

```bash
pnpm dev
```

Default ports:

- `knowledge-base` → `http://localhost:3000`
- `api` → `http://localhost:3001`
- `client` → `http://localhost:3002`
- `docs` → `http://localhost:3003`
- `landing` → `http://localhost:3004`

## Useful scripts

- `pnpm --filter client dev` – run only the admin dashboard.
- `pnpm --filter api dev` – run only the API.
- `pnpm --filter docs dev` – run the docs site.

## Project conventions

- UI is primarily built with shadcn and Tailwind.
- BlockNote provides rich-text editing.
- Most API routes live under `/api/v1/*` and are protected by JWT.
